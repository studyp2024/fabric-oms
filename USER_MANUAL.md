# 网络安全设备运维审计系统 - 用户手册

本手册旨在指导用户如何使用网络安全设备运维审计系统。系统分为 **管理员 (Admin)**、**运维人员 (Ops)** 和 **审计员 (Auditor)** 三种角色。

## 1. 系统登录

*   **访问地址**: `http://<服务器IP>/` (例如 `http://192.168.23.148/`)
*   **默认账号**:
    *   **管理员**: 用户名 `admin` / 密码 `admin`
    *   **运维人员**: 用户名 `ops` / 密码 `ops`
    *   **审计员**: 用户名 `auditor` / 密码 `auditor`

## 2. 功能概览

| 功能模块 | 管理员 (Admin) | 运维人员 (Ops) | 审计员 (Auditor) | 说明 |
| :--- | :---: | :---: | :---: | :--- |
| **服务器列表** | ✅ | ✅ | ❌ | 管理员可管理所有服务器，运维只能看到自己的服务器且无法查看密码。 |
| **WebSSH 终端** | ✅ | ✅ | ❌ | 点击按钮直接在网页端连接服务器进行操作。 |
| **审计日志** | ✅ | ❌ | ✅ | 查看所有服务器的操作日志，支持搜索和筛选。 |
| **用户管理** | ✅ | ❌ | ❌ | 管理系统用户，分配服务器给运维人员。 |

## 3. 详细操作指南

### 3.1 服务器管理与 WebSSH 连接 (Servers & WebSSH)
**路径**: 侧边栏 -> Servers

*   **查看服务器**: 列表显示所有已添加的服务器。管理员能看到所有服务器的 SSH 用户名和密码；运维人员只能看到分配给自己的服务器，**密码被系统安全隐藏**。
*   **添加服务器 (仅管理员)**:
    1.  在页面顶部的 "Add New Server" 表单中填写 IP、SSH端口 (默认22)、账号、密码。
    2.  点击 "Add Server" 保存。
*   **分配运维人员 (仅管理员)**:
    1.  在服务器卡片中找到 "Assign to" 下拉框。
    2.  选择一个运维人员 (Ops)，系统会自动保存关联。
*   **连接 WebSSH (核心功能)**:
    1.  点击服务器卡片下方的蓝色 **"Connect WebSSH"** 按钮。
    2.  系统会跳转到内置的黑色终端页面，自动建立 SSH 连接。
    3.  在此终端中输入命令，操作会实时记录并上链。点击 "Disconnect & Go Back" 即可安全退出。
*   **服务器状态监控**: 系统每分钟自动检查一次连通性，卡片右上角显示 ONLINE 或 OFFLINE。

### 3.2 审计日志 (Audit Logs)
**路径**: 侧边栏 -> Audit Logs (仅管理员和审计员可见)

*   **日志列表**: 实时显示从各服务器收集来的操作日志。
*   **日志字段**: Timestamp(时间), Server IP(来源), Client IP(操作者IP), User(SSH用户), Command(命令内容)。
*   **高危预警**: 包含敏感词 (如 `rm -rf`, `sudo` 等) 的命令会被智能合约拦截，并在前端以红色背景高亮显示 `⚠️ WARNING`。
*   **搜索与筛选**:
    *   顶部搜索框支持模糊搜索。
    *   勾选 "Only Sensitive" 可以只看高危命令。

### 3.3 用户管理 (User Management)
**路径**: 侧边栏 -> User Management (仅管理员可见)

*   **添加用户**:
    1.  在 "Add New User" 表单输入账号密码。
    2.  选择角色：**OPS** (运维人员) 或 **AUDITOR** (专职审计员)。
    3.  点击 "Add User" 保存。
*   **删除用户**: 点击用户列表右侧的 "Delete" 按钮（无法删除 admin 本身）。

## 4. 目标服务器日志配置

为了让系统能收集到目标服务器的操作日志，需要在**每台被纳管的目标服务器**上执行配置脚本。该脚本会配置 SSH 会话记录功能。

**注意**: 目标服务器**无需**克隆整个项目代码，只需获取配置脚本即可。

### 4.1 获取并上传脚本

1.  **获取脚本**: 在本项目代码库中找到 `scripts/setup_remote_logging.sh` 文件。
2.  **上传至目标服务器**:
    *   使用 WinSCP、FileZilla 或 `scp` 命令将该脚本上传到目标服务器的任意目录 (例如 `/tmp` 或 `/home/user`)。
    *   *示例 (使用 scp)*:
        ```bash
        scp scripts/setup_remote_logging.sh user@target-server-ip:/tmp/
        ```

### 4.2 执行配置脚本

登录到目标服务器，执行以下命令：

```bash
# 1. 进入脚本所在目录
cd /tmp

# 2. 使用 root 权限运行脚本
sudo bash setup_remote_logging.sh
```

**脚本执行内容说明**:
*   修改 `/etc/bash.bashrc`，添加 `PROMPT_COMMAND` 以实时记录所有用户的 SSH 命令。
*   创建日志文件 `/var/log/ssh_commands.log`。
*   设置权限为 `660` (root:users)，确保安全且可读。

### 4.3 验证配置

脚本执行完成后，新的 SSH 会话将自动生效。你可以通过以下方式验证：

1.  **重新登录 SSH** 或执行 `source /etc/bash.bashrc`。
2.  执行几条命令 (如 `ls`, `pwd`)。
3.  查看日志文件:
    ```bash
    tail -f /var/log/ssh_commands.log
    ```
    你应该能看到类似以下的日志记录：
    `2023-10-27 10:00:00 | SSH_IP:192.168.1.5 | USER:ops | PWD:/home/ops | COMMAND:ls -la`

## 5. 常见问题
*   **日志未更新**: 
    *   检查目标服务器是否执行了配置脚本setup_remote_logging.sh。
    *   检查审计系统的后端服务是否运行。
    *   确认数据库中配置的 SSH 账号密码正确且有权限读取日志文件。
